Sep of Concerns

What is Separation of Concerns?

separation of concerns (SoC) is a design principle for separating a computer program into distinct sections. Each section addresses a separate concern, a set of information that affects the code of a computer program.

Separation of Concerns


Purpose of SoC

It's crucial because the individual components' absence of duplication and singularity of purpose makes the entire system easier to manage. Separation of concerns is a technique for reducing connection between classes and increasing cohesion. The degree of interdependence between software components is referred to as coupling. The degree to which the pieces inside a module belong together is referred to as cohesion.

You can see a nice illustration of cohesiveness in the example below, where the class inherits values from the superclass. Because the program is performed in the module, this reduces coupling.

Example of Separation of Concerns from calculator

You can see separation of concerns in the example below.

class Subtraction(Calculation): def get_result(self): difference_of_values = 0.0 for value in self.values: difference_of_values = difference_of_values - value print.print(value) return difference_of_values

This is an example of separation of concerns from the calculator program because the calculator has separate files for addition, as well as subtraction, multiplication and division, instead of having all the code crammed up in a single file. This makes the code simpler and easier to understand
